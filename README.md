# Notes Bot — Telegram-бот для заметок

Простой и аккуратный бот для личных заметок в Telegram. Отправляешь любой текст — он сохраняется как заметка; в один тап можно посмотреть весь список и удалить ненужное.

Написан на **TypeScript** с разделением кода по слоям (конфиг / хранилище / бот) — как пример чистой структуры небольшого проекта.

## Возможности

- **Добавление заметки** — любой присланный текст сохраняется автоматически
- **Список заметок** — кнопка «Мои заметки» показывает всё, что сохранено
- **Удаление** — заметки удаляются одной кнопкой, по уникальному `id` (без «съезжающих» индексов)
- **Изоляция по пользователям** — у каждого свой список заметок
- **Аккуратное оформление** в едином монохромном стиле

## Стек

- [TypeScript](https://www.typescriptlang.org/)
- [Telegraf](https://telegraf.js.org/) — фреймворк для Telegram Bot API
- [Node.js](https://nodejs.org/) (long polling)
- [dotenv](https://github.com/motdotla/dotenv) — переменные окружения
- [https-proxy-agent](https://github.com/TooTallNate/proxy-agents) — работа через прокси

## Структура проекта

```
notes-bot/
├── src/
│   ├── bot.ts        # точка входа: обработчики и запуск бота
│   ├── config.ts     # загрузка токена и прокси из окружения
│   └── storage.ts    # хранилище заметок и операции над ними
├── .env.example      # пример конфигурации
├── tsconfig.json
└── package.json
```

Логика хранения вынесена из `bot.ts` в `storage.ts`: бот общается с пользователем, хранилище отвечает за данные. Это упрощает поддержку и позволяет позже заменить in-memory хранилище на базу данных, не трогая обработчики.

## Запуск

1. Клонировать репозиторий и установить зависимости:
   ```bash
   git clone <url>
   cd notes-bot
   npm install
   ```

2. Создать файл `.env` (по образцу `.env.example`) и указать токен бота от [@BotFather](https://t.me/BotFather):
   ```
   BOT_TOKEN=ваш-токен
   ```
   Если Telegram недоступен напрямую, можно задать прокси:
   ```
   HTTPS_PROXY=http://user:pass@host:port
   ```

3. Запустить в режиме разработки (авто-перезапуск при изменениях):
   ```bash
   npm run dev
   ```

   Или собрать и запустить продакшен-версию:
   ```bash
   npm run build
   npm start
   ```

## Команды

| Скрипт          | Действие                                      |
| --------------- | --------------------------------------------- |
| `npm run dev`   | запуск через `tsx` с авто-перезапуском        |
| `npm run build` | компиляция TypeScript в папку `dist/`         |
| `npm start`     | запуск собранной версии из `dist/`            |

## Заметки по реализации

- Заметки хранятся в памяти (`Map<userId, Note[]>`), поэтому сбрасываются при перезапуске. Для постоянного хранения можно подключить БД, изменив только `storage.ts`.
- Для удаления используется уникальный `id` заметки, а не её позиция в списке — список можно безопасно менять, не ломая кнопки.
